## Overview

The **AI Career Intelligence Platform** is a locally-hosted web application built with **FastAPI** (Python backend) and **Vanilla JavaScript** (frontend). It helps job seekers by:

1. Comparing their resume/CV against any job description
2. Detecting skill gaps and matched skills
3. Generating AI-powered, prioritized learning roadmaps
4. Scoring resumes against ATS (Applicant Tracking System) standards
5. Running mock technical interviews with honest, strict evaluation

The platform uses **Groq's API** for all AI language model tasks — fast, free-tier available, no local GPU required — and **Sentence Transformers** for semantic similarity scoring.

---

## Features

### 1. 📊 Resume Analyzer
Upload your CV and paste any job description. The platform returns:
- **Match Score** — percentage of job-required skills found in your resume
- **Semantic Score** — AI-computed overall content similarity (0–100)
- **Success Probability** — weighted prediction combining both scores
- **ATS Score** — how well your resume will pass automated screening
- **Score Breakdown** — separate bars for Skills, Formatting, and Keywords
- **Matched Skills** — skills you already have that the job needs
- **Missing Skills** — skills the job requires that are absent from your resume
- **AI Suggestions** — Groq-generated specific, actionable improvement tips
- **Smart Suggestions** — role-based required vs. missing skill analysis
- **Personalized Roadmap** — prioritized learning plan generated by AI

### 2. 🗺️ My CV Roadmap
Upload your CV and enter a target role or paste a full job description. The AI analyses your current skills against the role requirements and generates a structured roadmap with:
- Each skill ranked by **priority** (Critical / Important / Nice-to-have)
- Whether you already **have** or are **missing** each skill
- **Why** each skill matters specifically for this role
- A **3-step concrete learning plan** per missing skill
- **Focus First** badges on the top 3 most critical skills to tackle immediately

### 3. ✏️ Auto Fix Resume
Upload your resume and provide a job description. Groq rewrites your resume to:
- Add missing keywords for ATS optimization
- Improve bullet points with strong action verbs and measurable results
- Tailor content specifically to the job description

### 4. 🎤 Mock Interview
Practice real technical interviews:
- Enter any role (e.g., "Data Scientist", "Backend Developer")
- Get one challenging, role-specific technical question generated by Groq
- Type your answer (**copy-paste is disabled** — you must type your own answer)
- Receive honest, structured evaluation with Score, Strengths, Weaknesses, Model Answer, and Improvement Tip

### 5. 💬 AI Career Chatbot
Ask anything about your career — resume tips, salary negotiation, interview prep, career switches. The chatbot maintains conversation history (last 3 exchanges) for contextual responses.

## Installation & Setup

### Prerequisites

- **Python 3.9 or higher**
- **pip** (Python package manager)
- A **Groq API key** — get one free at [console.groq.com](https://console.groq.com)
- Internet connection (for Groq API calls)

### Step 1 — Extract the Project

```bash
unzip ai-career-updated.zip
cd ai-career-updated
```

### Step 2 — Create a Virtual Environment (Recommended)

```bash
python -m venv venv

# On Windows:
venv\Scripts\activate

# On macOS/Linux:
source venv/bin/activate
```

### Step 3 — Install Dependencies

```bash
pip install -r requirements.txt
```

> **Note:** The first time you run the app, `sentence-transformers` will automatically download the `all-MiniLM-L6-v2` model (~22MB). This is a one-time download and cached locally afterward.

### Step 4 — Set Your Groq API Key

Open `backend/.env` and set your key:

```env
GROQ_API_KEY=gsk_your_actual_groq_api_key_here
```

Alternatively, export it as an environment variable before running:

```bash
# macOS/Linux
export GROQ_API_KEY=gsk_your_key_here

# Windows (Command Prompt)
set GROQ_API_KEY=gsk_your_key_here

# Windows (PowerShell)
$env:GROQ_API_KEY="gsk_your_key_here"
```

## How Each Feature Works

### Resume Analyzer — `/analyze`

```
User uploads CV + job description
        ↓
parser.py        → Extract raw text from PDF / DOCX / TXT
smart_skills.py  → Extract skills from CV (3 layers: static + synonyms + Groq AI)
smart_skills.py  → Extract skills from job description (same 3-layer process)
matcher.py       → Set intersection → matched skills, missing skills, match_score
semantic.py      → Sentence Transformer cosine similarity → semantic_score (0-100)
ats_score.py     → Keyword density + section detection → ats_score + breakdown
roadmap.py       → Groq AI generates prioritized JSON roadmap
matcher.py       → Groq AI generates actionable suggestions per missing skill
success_predictor.py → (match_score × 0.6) + (semantic_score × 0.4)
        ↓
All results packaged into one JSON response → rendered in browser
```

### CV Roadmap — `/cv-roadmap`

The same flow as `/analyze` but uses `target_role` (e.g., "Data Scientist") instead of a full job description. The roadmap endpoint passes the full resume text and role context to Groq and requests a structured JSON array. Each skill item returned looks like this:

```json
{
  "skill": "machine learning",
  "status": "missing",
  "priority": "critical",
  "importance_reason": "Core technical requirement for a Data Scientist role.",
  "plan": "Complete Andrew Ng's ML course → Build 3 end-to-end projects → Compete on Kaggle",
  "focus_first": true
}
```

### Mock Interview — `/interview/question` + `/interview/evaluate`

**Question generation:** Groq receives the role name and returns one specific, challenging technical question. The prompt explicitly excludes generic soft-skill questions.

**Answer evaluation:** Before calling Groq, Python-level copy-paste detection runs:

```python
q_words = set(question.lower().split())
a_words = set(answer.lower().split())
overlap = len(q_words & a_words) / max(len(q_words), 1)
is_copy_paste = overlap > 0.7
```

The `is_copy_paste` flag, word count, and full answer are all passed to Groq with a strict scoring rubric that cannot be bypassed.

## Key Files Explained

### `groq_client.py` — Central AI Engine

Every utility that needs AI goes through this one file. This design means if you ever want to switch models or providers, you change exactly one file.

```python
def call_groq(prompt, max_tokens=1000, system="You are a helpful assistant.") -> str
```

Key design decisions:
- **No hardcoded API key** — reads from environment variable only, raises a clear error if missing
- **Prompt size capped at 4000 characters** — prevents HTTP 400 errors from oversized requests
- **Temperature 0.3** — lower randomness means more consistent, predictable output
- **Safe error returns** — timeouts, connection errors, and API errors all return a user-friendly warning string instead of crashing the server
- **Debug logging** — prints Groq status code and raw response to terminal for easy debugging

### `smart_skills.py` — 3-Layer Skill Extraction

| Layer | Method | Speed | Coverage |
|---|---|---|---|
| 1. Static DB | String matching against skills.json | Instant | 190 predefined skills |
| 2. Synonyms | Dictionary lookup (ml→machine learning) | Instant | 14 common abbreviations |
| 3. Groq AI | LLM extraction from raw text | ~1 second | Everything else |

The three-layer approach ensures both speed and completeness. Static matching catches known skills instantly. AI extraction catches niche tools, implied skills, and domain-specific terminology the static list would miss.

### `roadmap.py` — Structured Output from LLM

Getting a language model to return valid, parseable JSON consistently requires careful prompt engineering. The roadmap prompt specifies the exact JSON schema, all allowed field values, and rules for each field. To handle cases where the model adds extra text or markdown:

```python
# Strip markdown code fences if model adds them
if "```" in raw:
    for part in raw.split("```"):
        if part.strip().lstrip("json").strip().startswith("["):
            raw = part
            break

# Extract the JSON array by finding its boundaries
start, end = raw.find("["), raw.rfind("]")
roadmap = json.loads(raw[start:end + 1])
```

A template-based fallback activates if Groq fails or returns invalid JSON — the platform never crashes.

### `semantic.py` — Why Keywords Are Not Enough

Keyword matching alone misses semantic equivalents. "Build ML pipelines" and "develop machine learning systems" mean the same thing but share zero keywords. `all-MiniLM-L6-v2` encodes both texts into 384-dimensional vectors and measures the angle between them (cosine similarity). Texts with the same meaning point in similar directions in vector space.

### `db.py` + `memory.py` — Conversation Memory

SQLite stores chat history per `user_id`. The chatbot retrieves the last 3 exchanges and includes them in each prompt, enabling the AI to give contextual responses that reference earlier parts of the conversation.

## Scoring Systems

### Match Score
```
matched = resume_skills ∩ job_skills        (Python set intersection)
missing = job_skills − resume_skills        (Python set difference)
match_score = (len(matched) / len(job_skills)) × 100
```

### Semantic Score
```
model = SentenceTransformer('all-MiniLM-L6-v2')
score = cosine_similarity(model.encode(resume), model.encode(job_desc)) × 100
Range: 0 (completely different) to 100 (identical meaning)
```

### ATS Score
```
keyword_points = (matched_keywords / total_keywords) × 50   (max 50)
density_bonus  = +2 per skill that appears 2+ times in resume
section_bonus  = +5 per section present (skills / experience / projects / education)
ats_score      = min(sum of above, 100)
```

### Success Probability
```
success = (match_score × 0.6) + (semantic_score × 0.4)
```
Match score is weighted 60% because keyword presence is the primary ATS filter in real hiring systems.

## Mock Interview Rules

The Groq evaluation prompt enforces a strict rubric that the model must follow with no exceptions:

| Score | Trigger Condition |
|---|---|
| **0 / 10** | Copy-paste detected — answer word overlap with question is >70% |
| **0 / 10** | Answer is completely off-topic or irrelevant to the question |
| **1–2 / 10** | Fewer than 15 words, gibberish, or "I don't know" |
| **3–4 / 10** | Vague with no technical depth — surface-level definition only |
| **5–6 / 10** | Partially correct but missing key technical details |
| **7–8 / 10** | Clear, accurate, technically sound answer that addresses the question |
| **9–10 / 10** | Exceptional — thorough, includes examples, edge cases, and deep expertise |

Every evaluation response is structured into five sections:
1. **Score** — X/10
2. **Strengths** — what the candidate did well (or "None" for poor answers)
3. **Weaknesses** — specific technical gaps, errors, and missing depth
4. **What a Strong Answer Looks Like** — a model answer worth 9/10
5. **Tip to Improve** — one concrete, actionable suggestion

**Copy-paste is also blocked at the browser level** using `onpaste="return blockPaste(event)"` and `ondrop="return false"` on the answer textarea. A red warning message appears if a user attempts to paste.

## Troubleshooting

### `ValueError: GROQ_API_KEY not set`
The app cannot start without the API key. Add it to `backend/.env`:
```env
GROQ_API_KEY=gsk_your_key_here
```

### `ModuleNotFoundError` for any package
Make sure your virtual environment is activated, then reinstall:
```bash
pip install -r requirements.txt
```

### Sentence Transformers takes a long time on first run
Normal behaviour. The model downloads once (~22MB) and is cached locally at `~/.cache/huggingface/`. All subsequent runs start instantly.

### Skills show as "None detected"
Your CV may be a scanned image-based PDF with no text layer. The parser extracts text from the text layer only. Try converting your PDF to a text-based format, or copy-paste the content into a `.txt` file and upload that instead.

### `⚠️ AI service error` in the UI
- Verify your API key at [console.groq.com](https://console.groq.com)
- Check your Groq free-tier rate limits (30 requests/minute, 14,400/day for `llama-3.1-8b-instant`)
- Check the terminal — `groq_client.py` prints the exact status code and response body for every call

### Port 8000 already in use
```bash
# Find the process using port 8000
lsof -i :8000          # macOS / Linux
netstat -ano | findstr :8000   # Windows

# Or change the port in run.py:
uvicorn.run("backend.main:app", host="0.0.0.0", port=8001, reload=True)
```
